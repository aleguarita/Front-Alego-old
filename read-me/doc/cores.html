Cores

O Front-Alego™ trabalha com as cores de duas formas: através de uma paleta de cores que pode ser chamadas por nomes específicos (exemplo basica) e através de um tema que pode alterar todas as cores do sistema, sem precisar mudar a paleta, mantendo a harmonia.

As cores ganham variações clara e escura através de uma função baseada no Bourbon.

Todas as cores devem ser definidas aqui. Jamais chame uma cor em outro CSS usando hexadecimal, nome de cor ou variável!

Paletas

Neste arquivo você irá configurar as cores usadas pelo sistema. A paleta é dividida em três grupos: Cores reservadas, que são cores padrão para todos os sistemas e não devem ser alteradas; Cores do sistema que formam o tema do sistema atual e Cores de temas específicos que são cores voltadas para diversos temas fixos.

Cada cor representa uma matiz principal e deve ter um nome fácil de lembrar, com um comentário indicando qual a cor (apenas para facilitar a lembrança, é possível ver todas as paletas aqui). Sempre comece alterando as já estabelecidas e acrescente novas conforme a necessidade.

Segue a lista indicativa de matizes criadas até o momento:

Cores reservadas

cinza
É o cinza médio. Com ele é criado a paleta de tons de cinza. Para chamar branco utilize paleta(cinza, claro, 100) e para chamar preto utilize paleta(cinza, escuro, 100)
erro
Vermelho indicativo de erro ou urgência. Não altere.
ok
Verde indicativo de sucesso. Não altere.
alerta
Amarelo indicativo de aviso ou alerta. Não altere.

Cores de temas específicos

verde-bandeira
Verde da bandeira de Goiás e do Brasil. Não altere. Se precisar de outro tom de verde crie uma paleta separada
amarelo-bandeira
Amarelo da bandeira de Goiás e do Brasil. Não altere. Se precisar de outro tom de amarelo crie uma paleta separada
azul-bandeira
Azul da bandeira de Goiás e do Brasil. Não altere. Se precisar de outro tom de azul crie uma paleta separada. Este azul também é usado no tema "azul"
rosa
Rosa para o tema do Outubro Rosa Não altere. Se precisar de outro tom crie uma paleta separada

Cores do sistema

basica
A cor básica/principal do sistema. Altere esta no seu sistema como cor principal.
secundaria
A cor secundária/destaque do sistema. Altere esta cor no seu sistema como cor secundária.

Temas

Depois das paletas o arquivo lista os temas de cor. Você pode criar vários outros temas e alterar no arquivo de configuração qual o tema atual. Ao escolher um tema, as cores do sistema mudam de acordo.

Assim como acontece com a $paleta, os temas estão agrupados dentro de um mapa chamado $temas. Dentro deste mapa teremos um sub-mapa para cada tema, que são (por padrão):

principal
O tema padrão do sistema. Se você não for usar outro tema, basta alterar somente este.
cinza
Este tema faz uso apenas de tons de cinza. Pode ser útil em um momento de prototipação e definição de contrastes
alto-contraste
Um tema com cores bem contrastantes, pensando em acessibilidade.
bandeira
Faz uso das cores da bandeira de Goiás. Para utilizar em sistemas mais institucionais ou eventos do Estado.
rosa
Tema com predominância das cores rosa. Para o Outubro Rosa, por exemplo.
azul
Tema com predominância das cores azuis. Para o Novembro Azul, por exemplo.

Acrescente novo mapa conforme a necessidade. Duplique o mapa principal e trabalhe nele (um exemplo de possibilidade para isso é um tema de Natal).

Dentro de cada tema existem diversos mapas com suas cores específicas. Podemos dividir estes mapas em três categorias: elementos genéricos (que são usados em qualquer contexto), elementos específicos (que são usados dentro de um contexto específico e já está sendo sendo usado dentro das configurações do Front-Alego, em arquivos próprios. Ex.: tabela) e elementos do sistema que são os elementos criados para o seu sistema (exemplo: menu, cabeçalho). Cada uma destas categorias estão separadas por um comentário com seu nome.

Sempre que você precisar de uma cor para um elemento não declarado, crie este mapa dentro de CADA tema abaixo do comentário //elementos do sistema. Acrescente as cores dentro deste mapa conforme a necessidade.
Dentro de cada mapa a cor padrao será chamada pela função cor() sem que se especifique qual é. Use, portanto, para a cor principal daquele elemento.
texto
Elemento genérico. Cores para serem usadas em textos. padrao, claro, medio, destaque, titulo
fundo
Elemento genérico. Cores para serem usadas em fundos. padrao, claro, x-claro, escuro,sombra (cor da sombra, não é um fundo, mas achei melhor colocar aqui e evitar uma categoria própria).
borda
Elemento genérico. As cores das bordas. Lembre-se que se não declarar a cor da borda ela será, por padrão, a do texto. padrao, variante.
link
Elemento específico. Define as cores do link, com três variações. padrao, hover; escuro, escuro-hover; claro e claro-hover
formulario
Elemento específico. Cores do formulário. padrao, placeholder (usado no texto do placeholder do formulário), ativo (usado quando o formulário está focado ou com o mouse em cima), fundo.
botao
Elemento específico. Duas cores usadas nos botões. padrao e secundario. Atenção: antes os botões usavam ainda mais quatro cores e na versão 1.2.0 do Front-Alego foi simplificado. No entanto, se você usar as cores antigas, não haverá problema de compatibilidade.
tabela
Elemento específico. Elementos das tabelas. borda, borda-clara, borda-escura, fundo, titulo
lista
Elemento específico. Listas formatadas. padrao, clara, escura
modal
Elemento específico. Cores das modais. fundo, titulo-fundo (formata a cor de fundo do título), titulo-texto
paginacao
Elemento específico. Cores da paginação. padrao, clara, escura
breadcrumbs
Elemento específico. Cores dos breadcrumbs. link, atual
card
Elemento específico. Elementos dos cards. fundo, borda, titulo, destaque
secao-lista
Elemento específico. Blocos de conteúdo que podem ser ocultados, virando uma lista. padrao, clara
abas
Elemento específico. As cores da abas. padrao, inativa e fundo-inativo

Inserindo cores dentro do tema

Para chamar as cores dentro dos temas, você fará através da função abaixo (nunca chame através de hexadecimal ou variável):

paleta()
Esta função é para chamar as cores que você criou na $paleta. Para chamar a cor pura (aquela que você definiu na paleta), escreva paleta(nome-cor). Para chamar as variações claras, chame com paleta(nome-cor, clara, $valor). O $valor pode ir de 1 a 100. Para chamar cores escuras chame com paleta(nome-cor, escura, $valor)

Inserindo as cores no SASS

Como explicado anteriormente, você só poderá usar as cores definidas nos temas. Para chamar uma cor, use uma das duas funções abaixo:

cor-status()
Para chamar uma das cores de status. Não use para formatação de interface, somente use para status reais. Exemplo: você quer alertar que o campo preenchido está com erro. color: cor-status(erro). Esta função trabalha igual à função paleta() e aceita variantes claro e escuro
cor()
Principal função para chamar as cores na interface. Cada elemento tem um conjunto de cores a ser chamado, além dos genéricos (conforme já explicado anteriormente). Exemplo: seu texto vai ser um texto destaque. cor(texto, destaque). A cor padrao de cada conjunto não precisa ser chamada, ou seja, o texto padrão seria: cor(texto). Essa função não tem variantes de claros/escuros para cada cor, justamente para manter a uniformidade. Se você realmente precisar, recomendo as funções tint e shade do Bourbon

Se acrescentar mais cores, abra o arquivo read-me/elementos.html e duplique a lista e acrescente o modificador --nome-paleta. Depois abra o read-me/info.scss e acrescente o nome da paleta no mapa $cores-paleta (e recompile o arquivo).
