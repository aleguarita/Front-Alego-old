Configurações básicas

config/_+config.scss

O arquivo de configuração permite escolher a formatação básica do seu projeto. Em muitos casos você escrevará o valor a ser usado. Se o valor for "sim" ou "não" o valor relevante é o sim e qualquer valor diferente é entendido como não.

Uso de Classes

O mapa $uso-classes define se vai criar algumas classes no CSS final. Caso não vá usar alguma função comente o nome dentro do mapa (cmd + /), isso vai fazer com que as classes não apareçam no arquivo CSS compilado, deixando-o mais leve (especialmente útil em sites externos).

As classes criadas nos arquivos adicionais na pasta extra sempre aparecerão, mesmo que você comente as classes criadas pelo @uso-classes. Isso significa que se você criar uma tabela personalizada somente a sua tabela vai aparecer no CSS final.

form
Formatação dos tags básicos de formulário, incluindo sua relação entre label e input. Inclui formatação de campos de textos (e derivados dos HTML 5), caixa de texto, radiobutton, checkbox e select.
form-pre
Espaçamento entre os formulários. Cada pre-x dá o mesmo espaço de uma "grid" de formulário. Por ser algo de uso raro, por padrão já vem comentado
form-grupo
Grupo de formulários. Serve para definir a largura deles, com um sistema similar ao de grids (com classes de grid-1 até grid-12)
form-grupo-responsivo
Grupo de formulários responsivos. Se a classe anterior (form-grupo) também estiver comentada, essa não ira aparecer.
botao
Botões de formulário
botao-secundario
Caso seja necessário dois tipos diferentes de botão
autocomplete
Formulário de autocomplete
grid-container
As classes de container de grid
tabela
Classe de tabela
tabela-form
A classe secundária .tab--form. Se a tabela (acima) estiver comentada, esta não será escrita.
modal
Classe da modal
card
Classe dos cards
lista
Classes para a formatação de listas (o menu deste arquivo faz uso)
paginacao
Formata a paginação
breadcrumb
Formata o breadcrumb
toaster
Formata os toasters do sistema
secao-lista
Seções que podem ser fechadas, ocultando o conteúdo e mostrando o título, formando uma lista de seções
abas
Formata as abas (e inclui a exibição/ocultamento do conteúdo)
tela-mensagem
Classe para a criação de telas de mensagens curtas (como erro 404 ou de permissão negada)
simbolos-svg
Inclusão das classes de símbolos de SVG
simbolos
Símbolos por fontes. Está aqui por retrocompatibilidade (deixe comentado para simplificar o código)
sprite
Arquivos de sprites
datepicker
Formata o datepicker, se estiver em uso
fonte-serifa
Define se usa fontes personalizadas serifadas (patua one)
fonte-sans-serifa
Define se usa fontes personalizadas sem serifa (open-sans)
Apenas comente! Não altere os nomes em hipótese nenhuma (do contrário será o mesmo efeito de comentar e a classe não será criada)

Tema de cores

Defina qual o tema de cor utilizado. Veja mais detalhes sobre isso em cores. Abaixo uma explicação dos valores aceitos em $cores-tema e seus possíveis usos:

principal
O tema padrão e definido originalmente para o sistema
monocromatico
Faz uso de apenas a paleta principal do sistema
cinza
Utiliza apenas tons de cinza
auto-contraste
Tons escuros e claros em auto contraste, para acessbilidade
bandeira-goiana
Faz uso das três cores da bandeira goiana
rosa
Usa principalmente tons de rosa. Para o Outubro Rosa
azul
Usa principalmente tons de azul. Para o Novembro Azul

Caso você crie algum tema novo, basta alterar a variável $cores-tema para o tema novo.

Grid

Defina aqui os detalhes da grid. Veja mais detalhes sobre isso em grids.

$grid-qtd
O maior valor de grid (grid-#) disponível. Por padrão são 12 grids (melhor valor de se trabalhar por ser divísivel por 3 e 4).
$grid-margem
A margem entre as grids. Por padrão é o equivalente a 10px convertidos para rem. Atenção: afeta a variável $calha
$grid-max
O tamanho máximo da grid na tela (max-width). Quando a tela for maior que essa medida, fica centralizado (usa no .container)

Tabelas

Configurações de tabela.

$tab-tipo
Tipos de tabela: tipo-1 | tipo-2
$tab--zebra
Determina se a tabela é zebrada ou não: sim | não
$tab--responsiva
Determina se a tabela é responsiva ou não: sim | não
$tab--responsiva-tipo
Se a tabela for responsiva determina se vai usar media query de tamanho (min-width) ou de orientação da tela (orientation: portrait): tamanho | orientação
$tab-acao-link-bloco
O ícone de ação aparece como um bloco, possibilitando o hover ocupar uma área maior
$tab-col--maximo
O valor máximo do placeholder criado para determinar as larguras das colunas. O valor atual é , o que quer dizer que a maior coluna vai ter px (devidamente convertido para rem)
$tab-resp-tamanho
Determina o tamanho mínimo para ser responsiva. O valor é baseado nas medidas dos breakpoints (ver veja mais detalhes aqui): pequeno | medio
$largura-tabela-col
Mapa de classes para determinar a largura das colunas. Coloque o nome e valor e separe por vírgula. Exemplo: nome: 10,. O valor deve ser em pixels mas sem unidade.
$largura-tabela-col-min
Mapa de classes para determinar a largura mínima das colunas.
$largura-tabela-col-max
Mapa de classes para determinar a largura máxima das colunas.

Paginação

Configuração da paginação

$paginacao-tipo
Tipos de paginação: tipo-1 | tipo-2
$paginacao-posicao
Define o alinhamento (se aparece à direita, centralizado ou à esquerda). Use os valores de flex-box: flex-start (à esquerda) | center | flex-end (à direita)
$paginacao-fonte
O tamanho da fonte da paginação.

Modal

Define as configurações da modal.

$modal-tipo
Define o tipo da paginação: tipo-1 | tipo-2
$modal-transparencia
Define qual a transparência do fundo da modal (que tampa o conteúdo do site). Quanto maior o valor, mas tranparente. O valor vai de 0 (totalmente opaco) até 1 (totalmente transparente)

Card

Define a configuração dos cards.

$card-tipo
Tipo de card: tipo-1 | tipo-2

Toaster

Configurações do Toaster.

$toaster-tipo
Tipo de toaster: tipo-1 | tipo-2
$toaster-horizontal
Define se vai aparecer à esquerda ou à direita da página. Valores: right | left
$toaster-vertical
Define se o toaster vai aparecer de cima ou de baixo. Valores: top | bottom
$toaster-distancia
Define a distância que começa da extremidade do browser. Use medidas em ou rem
$toaster-distancia-vertical
Um multiplicador para a distância vertical. Isso é usado para distanciar o toaster do rodapé (se aparecer de baixo) ou do cabeçalho (se estiver vindo de cima) sem ocultar estes elementos.

Formulários

Configuração dos formulários

$form-tipo
Tipo de formulário: tipo-1 | tipo-2 | tipo-3
$label-tipo
Define se o label vai ser na mesma linha do input ou se vai ter quebra de linha. Valores: flutuante (quebra de linha) | colado (na mesma linha)
$form-radio-check-personaliza
Determina se vai usar formatação personalizada nos radio button e checkbox ou se vai usar o padrão do browser. Valores sim | não
$form-obrigatorio
Define o tamanho do asterisco de obrigatório dentor dos campos. A medida é o equivalente em pixels mas deve ser escrita sem a medida
$botao-tipo
Determina o tipo de botão a ser usado. Valores: tipo-1 até tipo-6
$botao-secundario
Determina o tipo secundário de botão. Se for o mesmo tipo do acima, aparecerá com outra cor. Valores: $botao-tipo ou tipo-1 até tipo-6
$botao-sombra-colorida
Define se a sombra vai ser colorida, da cor do texto. sim | não
$botao-arredondamento
Define se o botão vai ter borda arredondada. sim | não
$botoes-status
Não crie classes neste mapa (pois vai dar erro), apenas comente as que não for usar. Usam o mesmo tipo do $botao-secundario

Sprites

Definições dos sprites. Cria várias classes conforme o nome que você atribuir.

$sprite-arquivo
Determina qual o nome do arquivo usado. Valor padrão: png
$sprite-medida
Determina a medida horizontal e vertical de cada imagem. Valor padrão: 150px
$sprites
Mapa que cria as classes. Coloque o nome da classe desejada e entre com os valores das coordenadas x, y. Caso a imagem seja maior ou menor que o padrão coloque o multiplicador logo depois, separados por vírgula (ex.: 2, .5; neste caso a imagem tem o dobro da largura e metade da altura). A altura pode ser omitida se ela for igual à largura. Veja aqui mais detalhes de uso.
