Mixins

Segue uma lista resumida dos mixins. Para maiores detalhes de como usar alguns tem descrição com exemplo de uso. Mixins que por ventura não esteja dentro do arquivo base/_mixins.scss podem ser editados por você e estarão identificados como Mixin editável.

Mixins de configuração. Para configurar uma parte específica

prefixo(declaração, prefixos)
Este mixin é para ser usado na criação de outros mixins que simplifiquem o acréscimo de vendor prefix. Diversas propriedades de CSS3 já tem prefixos tanto no Front-Alego como no Bourbon
media-tipo(breakpoint, medida)
Este mixin chama a media query pensando na técnica mobile first (utilizando min-width) como Valor padrão. O breakpoint é um dos descritos aqui

Mixins de vendor prefixes. Para usar propriedades de CSS que dependem de prefixos de browser.

placeholder
Para formatar o elemento placeholder dos inputs de texto
transform(valor)
Aplica os prefixos para os transform de CSS3
transform-origin(valor)
Aplica os prefixos para os transform-origin de CSS3
display-flex(largura)
Dá o display: flex para o seletor. O valor $largura é opcional e você determina a largura do elemento pai que está recebendo o display: flex;
flex-direction(valor)
Aplica a propriedade flex-direction no seletor. O valor padrão do mixin é column
flex-wrap(valor)
Aplica a propriedade flex-wrap no seletor. O valor padrão do mixin é wrap
flex-flow(direction, wrap)
Aplica a propriedade composta flex-flow. O valor padrão do mixin é column, wrap
align-items(valor)
Aplica a propriedade align-items no seletor. O valor padrão do mixin é center
align-content(valor)
Aplica a propriedade align-content no seletor. O valor padrão do mixin é center
order(valor)
Aplica a propriedade order no seletor. O valor padrão do mixin é auto
flex-grow(valor)
Aplica a propriedade flex-grow no seletor. O valor padrão do mixin é 1
flex-shrink(valor)
Aplica a propriedade flex-shrink no seletor. O valor padrão do mixin é 1
flex-basis(valor)
Aplica a propriedade flex-basis no seletor. O valor padrão do mixin é 100%
flex(grow shrink basis)
Aplica a propriedade composta flex no seletor. O valor padrão do mixin é 1. Os elementos não são separados por vírgula.
align-self(valor)
Aplica a propriedade align-self no seletor. O valor padrão do mixin é auto

Mixins de formatação. Estes mixins serve para incluir formatação dentro de algum seletor. Seja configurações estáticas (que seriam de placeholders), quanto dinâmicas, com valores variáveis.

clearfix
Aplica o hack de css de clearfix, para quando o elemento pai tiver float. Dispensa propriedades.
centraliza-caixa(largura, altura)
Cria uma caixa centralizada no meio da tela (ou do elemento pai, se ele tiver algum valor de position). Os valores de largura e altura são opcionais
grupo-flex(qtd, margem)
Deve ser usado num elemento pai (que ganha a propriedade display: flex). distribui os elementos filhos em tamanho igual. Use isso para fazer um grupo de cards, por exemplo.
Valores padrão: $qtd não tem valor padrão (é um mapa e aceita quatro valores); $margem: $calha a $calha foi definida no config/variaveis
texto-selecionado(fundo, texto)
Formata o seletor para quando o texto for selecionado no browser. Valores padrão: fundo: cor(fundo, escuro) e texto: cor(texto, claro). Dica: selecione o texto para ver os valores padrão em uso.
item-checado(tag)
Este mixin oculta o checkbox e permite formatar o elemento seguinte (tag) quando o checkbox estiver marcado. Normalmente utilizado para fazer listas que exibem/ocultam o conteúdo.

Mixins de tags específicos. Estes mixins devem ser usados dentro de tags específicos (como links, títulos, etc).

lista-sem-marcador()
Formata listas para serem sem marcadores. Ele tem um valor opcional (para omitir, não precisa colocar os parênteses). O valor padrão é null
texto(mod)
Mixin editável. Mixin para formatar textos padrão do site). Você pode alterar os valores padrões para melhor adequar ao sistema, mas prossiga com cautela. Também é possível acrescentar novos valores.
titulo(mod)
Mixin editável. Mixin para formatar classes de título.
link(mod)
Mixin editável. Mixin para formatar links.

Este mixin acrescenta prefixo de browser no CSS3.

Sintaxe: @include prefixo(($declaracoes), $prefixos) { }

$declaracoes
Sempre deve aparecer dentro de parênteses e cada declaração (se tiver mais de uma) separadas por vírgula. Exemplo: transform: translate(180deg)
$prefixos
Os prefixos usados. Separe com espaço. Exemplo: webkit moz
Para facilitar a leitura separe os blocos de elementos com enter

Exemplos de uso:

SASS

.seletor {

@include prefixo((column-count: 3, column-row: 1.5em), webkit moz);

}

// Para melhor leitura, separece com ENTER:

.seletor {

@include prefixo((

column-count: 3,

column-row: 1.5em),

webkit moz);

}

Isso vai resultar no código:

CSS

.seletor {

-webkit-column-count: 3;

-moz-column-count: 3;

column-count: 3;

-webkit-column-row: 1.5em;

-moz-column-row: 1.5em;

column-row: 1.5em;

}

Este mixin serve para determinar propriedades responsivas, utilizando mobile-first.

Sintaxe: @include media-tipo($breakpoint, $medida)

$breakpoint
Qual o breakpoint a ser usado. Não coloque um valor em números e sim um dos valores disponíveis no mapa $breakpoints. Veja aqui quais são os valores
$medida
Defina se a media query vai ser no máximo ou no mínimo aquele tamanho. Valores: min | max. Valor padrão: min (por padrão este mixin trabalha com mobile first).

Exemplo de uso:

SASS

.seletor {

@include media-tipo(medio) {

/* insira aqui a configuração */

}

}

Compilado:

CSS

@media (min-width: ) {

.seletor {

/* insira aqui a configuração */

}

}

Formata o placeholder de formulário, utilizado nos inputs. Este mixin não tem variáveis, coloque a formatação direta.

Sintaxe: @include placeholder { regras }

SASS

input[type="text"] {

@include placeholder {

color: cor(formulario, placeholder);

}

}

Vai resultar em:

CSS

input[type="text"]::-webkit-input-placeholder {

color: #40d4da;

}

input[type="text"]::-moz-placeholder {

/* Firefox 19+ */

color: #40d4da;

}

input[type="text"]:-ms-input-placeholder {

color: #40d4da;

}

transform()

Aplica os prefixos de CSS para a propriedade transform. Apesar de existir um mixim similar no Bourbon, o dele coloca muito prefixo, o do Front-Alego é mais atual e enxuto.

Sintaxe: @include transform($transform)

$transform
Qual o valor a ser aplicado. Ver no w3school

Veja um exemplo:

SASS

.seletor {

@include transform(translate(20px,20px));

}

Este é o resultado:

CSS

.seletor {

-webkit-transform: translate(20px, 20px);

transform: translate(20px, 20px);

}

transform-origin()

Aplica os prefixos de CSS para a propriedade transform-origin. Apesar de existir um mixim similar no Bourbon, o dele coloca muito prefixo, o do Front-Alego é mais atual e enxuto.

Sintaxe: @include transform-origin($origem);

$origem
Qual a origem da transformação. Ver no w3school.

Veja um exemplo:

SASS

.seletor {

@include transform-origin(right 0);

}

Este é o resultado:

CSS

.seletor {

-webkit-transform-origin: right 0;

transform-origin: right 0;

}

No elemento pai (flex-container)

display-flex

Coloca o elemento com display: flex. Deve ser no elemento pai. Este mixin não precisa de parâmetros. Saiba mais sobre isso.

Sintaxe: @include display-flex; ou @include display-flex($largura);

SASS

.seletor {

@include display-flex;

}

Compilado:

CSS

.seletor {

display: -webkit-flex;

display: -ms-flexbox;

display: flex;

}

Alternativamente você pode chamar uma largura para o elemento pai. Para fazer isso, use o valor opcional de $largura.

SASS

@include display-flex(100%);

Compila em:

CSS

.seletor {

display: -webkit-flex;

display: -ms-flexbox;

display: flex;

width: 100%;

}

Este mixin serve para aplicar o hack de css clearfix. utilize ele no elemento pai que usem float. Ou nem use, afinal... pra quê usar float mesmo?

Sintaxe: @include clearfix

Este mixin não tem valor a ser defindo basta chamar ele pelo nome:

SASS

.clearfix {

@include clearfix;

}

Este mixin cria uma caixa centralizada na tela com posicionamento absoluto (se o elemento pai tiver algum position definido vai centralizar com relação à ele). Ideal para criar uma tela de login, por exemplo. Não precisa usar em modais, pois este elemento já foi criado dentro do Front-Alego. Esta caixa não precisa ter tamanho fixo.

Sintaxe: @include centraliza-caixa($largura, $altura);

$largura
Define a largura da caixa. Este valor pode ser omitido.
$altura
Define a altura da caixa. Este valor pode ser omitido. Se for usar este valor, mas sem a altura, lembre-se de escrever o nome da variável.

Chamando o mixin sem os valores:

SASS

.sem-valor {

@include centraliza-caixa;

}

Chamando o mixin com os valores:

SASS

.com-medidas {

@include centraliza-caixa(35vw, 50vh);

}

Chamando o mixin com o valor de largura:

SASS

.so-largura {

@include centraliza-caixa(35vw);

}

Chamando o mixin com o valor de altura:

SASS

.so-altura {

@include centraliza-caixa($altura: 50vw);

}

Este mixin serve para criar itens de mesmo tamanho distribuídos ao longo de algum container. Ele deve ser aplicado no container (que se torna display: flex). Este mixin aceita o uso de conteúdo dentro dele, para configurar o elemento filho.

Sintaxe: @include grupo-flex($quantidade, $margem);

Ou:

Sintaxe: @include grupo-flex($quantidade, $margem) { @conteúdo }

$quantidade
A quantidade de caixas por linha. Sem valor padrão. Este valor é um mapa, cada valor do mapa (de até 4 números) indica uma medida dentro do valor responsivo, na ordem: pequeno, médio, grande e extra-grande. Se você quiser que tenha a mesma quantidade de elementos idenpendente do tamanho da tela, basta indicar o valor, caso especifique mais de um, é necessário que os valores estejam dentor de parênteses (ex.: (1, 3)).
$margem
Determina a margem entre os elementos. Se não quiser nenhuma margem, coloque o valor como 0. Valor padrão: 10px convertidos para rem.
@conteúdo
Escreva os atributos de CSS como se o mixin fosse ele mesmo um seletor. Estes atributos são do elemento filho (de cada uma das caixas).

Colocando 5 itens por linha:

SASS

.cinco-itens-linha {

@include grupo-flex(5);

}

Colocando 5 itens por linha, com margem de 1rem:

SASS

.grupo-itens {

@include grupo-flex(5, 1rem);

}

Colocando 1 item por linha em tela pequena e 2 itens por linha em telas maiores:

SASS

.grupo-itens {

@include grupo-flex((1, 2));

}

Colocando 1 item por linha em tela pequena e 2 itens por linha em telas maiores e margem de 1rem:

SASS

.grupo-itens {

@include grupo-flex((1, 2), 1rem);

}

Colocando 1 item por linha em tela pequena e 2 itens por linha em telas médias, 3 em telas grandes (tablet horizontal) e 5 em telas maiores (desktop):

SASS

.grupo-itens {

@include grupo-flex((1, 2, 3, 5));

}

Colocando 1 item por linha em tela pequena e 2 itens por linha em telas médias, 3 em telas grandes (tablet horizontal) e 5 em telas maiores (desktop) e margem 0:

SASS

.grupo-itens {

@include grupo-flex((1, 2, 3, 5), 0);

}

Como no de cima, mas com formatação do elemento filho (no caso, a inclusão de um padding e uma cor de fundo):

SASS

.grupo-itens {

@include grupo-flex((1, 2, 3, 5), 0) {

padding: 1rem;

background: cor(fundo);

}

}

Este mixin altera a forma como o browser apresenta o texto quando o usuário seleciona o texto (experimente selecionar um parágrafo desta documentação, que usam os valores padrão).

Sintaxe: @include texto-selecionado($fundo, $cor-texto);

$fundo
Define a cor de fundo. Valor padrão: cor(fundo, escuro)
$texto
Define a cor do texto. Valor padrão: cor(texto, claro)

Como você pode notar, é possível aplicar o mixin sem valores e as cores mudam conforme as cores aplicadas nos temas de cor.

Este mixin pode ser aplicado em todo o site (no <body>) ou em elementos específicos (como um parágrafo, uma div e assim por diante).

Exemplo de uso no body sem valores:

SASS

body {

@include texto-selecionado;

}

Exemplo de uso em um caso específico:

SASS

.urgente {

font-weight: bold;

color: cor-status(urgente);

@include texto-selecionado(cor-status(urgente-escuro), cor(texto, claro));

}

Que compila em:

CSS

.urgente {

font-weight: bold;

color: #ed1c24;

}

.urgente::selection {

background-color: #be1e2d;

color: #fff;

}

.urgente::-moz-selection {

background-color: #be1e2d;

color: #fff;

}

Este mixin faz uso da técnica de CSS .checkbox:checked ~ .label para formatar o elemento seguinte a um checkbox oculto quando o checkbox está marcado. Um exemplo disso é um conteúdo que está oculto, exibindo apenas o título e quando clica mostra o conteúdo (como usado em Seção Lista).

Sintaxe: @include item-checado($tag) { conteúdo }

$tag
A tag do conteúdo a ser formatado. Normalmente uma classe. Deve aparecer entre aspas

O HTML deve seguir o padrão:

  • Checkbox e conteúdo devem estar no mesmo nível hierárquico;
  • Depois do checkbox aparece o label (também no mesmo nível hierárquico);
  • Cada checkbox deve ter um id e este deve ser único. O label deve ter a propriedade for com o mesmo id do checkbox;
  • A classe do conteúdo não precisa ser diferente para cada item/lista (melhor que seja a mesma, inclusive, para deixar o CSS compilado menor);
  • O conteúdo vem aberto por padrão. Para ocultar o conteúdo daquele item, inclua checked="checked" no checkbox.

Exemplo de aplicação:

HTML

<div>

<input type="checkbox" class="check-lista" id="nome" />

<label for="nome">Título do item da lista</label>

<div class="conteudo-demo">

...

</div>

</div>
SASS

.check-lista {

@include item-checado(".conteudo-demo") {

display: none;

}

}

Neste exemplo, ao clicar no label, o conteúdo é ocultado.

Este mixin retira os marcadores de uma lista (ul), para utilizar em menus, listas com formatações especiais, etc. O valor deste mixin é opcional

Sintaxe: @include lista-sem-marcador($flex);

$flex
Determina se o elemento tem display: flex ou não. Valor opcional. Valor padrão: nenhum

Chamando sem valor padrão (apenas retira os marcadores):

SASS

.menu {

@include lista-sem-marcador;

}

Chamando com flex:

SASS

.menu-flex {

@include lista-sem-marcador(flex);

}

config/_tipografia.scss

Este mixin formata os textos padrão do sistema e já está sendo usado para formatar. Você pode alterar os valores (por exemplo, a margem). Mas fique atento que isso alterará todos os textos e títulos do site.

Sintaxe: @include texto($mod);

Valores que podem ser usados em $mod (estes valores devem ser escritos como demonstrado, do contrário vai retornar um erro):

padrao
Formata os blocos de texto padrão (<p>, <ul> e outros).
titulo
Formata os títulos do sistema (<h1> até <h6>)

Chamando os padrões de texto:

SASS

p {

@include texto(padrao);

}

Chamando os padrões de título:

SASS

h1 {

@include texto(titulo);

}

Alterarnativamente você pode criar um conjunto novos de regras com seu próprio nome, para isso acrescente o seguinte código antes da última chave ser fechada (}):

SASS

// COMENTÁRIO CURTO FALANDO O QUE É

@else if $mod == nome-novo {

// suas regras

}

config/_tipografia.scss

Este mixin serve para criar configurações especiais para os títulos. Eles devem ser usados em classes (ou com seletor descendente: .classe > h1) e nunca diretamente no tag, pois isso vai afetar todos os tags.

Você pode alterar totalmente os valores e acrescentar novas opções suas.

Sintaxe: @include titulo($mod)

Valores que podem ser usados em $mod (estes valores devem ser escritos como demonstrado, do contrário vai retornar um erro):

borda
Acrescenta uma borda inferior fina no título, com a mesma cor do mesmo

// COMENTÁRIO CURTO FALANDO O QUE É

@else if $mod == nome-novo {

config/_tipografia.scss

Este mixin serve para criar configurações especiais para os links. Eles devem ser usados em classes (ou com seletor descendente: .classe > a) e nunca diretamente no tag, pois isso vai afetar todos os links do sistema.

Algumas opções podem ser alteradas e outras não (abaixo indico quais não devem ser alteradas e também há um comentário no próprio mixin). Você também pode acrescentar novas opções.

Sintaxe: @include link($mod);

Valores que podem ser usados em $mod (estes valores devem ser escritos como demonstrado, do contrário vai retornar um erro):

sem-sublinhado
Retira do sublinhado do link e coloca quando passa ou mouse por cima. Não altere
bloco
Altera o link para que ele seja bloco (display: block) ocupando todo o elemento pai (evite colocar junto de outro elemento). Serve para fazer menus. Não altere
borda
Retira o sublinhado do link e acrescenta uma borda cinza na parte inferior. Quando passa o mouse por cima a borda fica da cor do link
pontilhado
Retira o sublinhado do link e substitui por uma borda pontilhada e cinza. Quando passa o mouse por cima o link escurece e a borda fica da mesma cor

Para retirar o sublinhado:

SASS

.link--padrao {

@include link(sem-sublinhado);

}

Para colocar em bloco:

SASS

li > a {

@include link(bloco);

}

Para fazer o link com borda:

SASS

.link--borda {

@include link(borda);

}

Para incluir a borda pontilhada no link:

SASS

.link--pontilhado {

@include link(pontilhado);

}
