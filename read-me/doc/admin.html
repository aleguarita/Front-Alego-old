Admin

admin.scss modelos/login.html

A view de admin se divide em dois arquivos HTML que podem ser encontrados na pasta modelos: login.html e admin.html. Evidentemente o Ruby fará uso de outros modelos, mas ajuste o mesmo para que as classes sejam as mesmas.

O arquivo básico para trabalhar ( admin.scss) está na raíz do FRONT-ALEGO e já está pré-formatado. Sinta-se à vontade para alterar as cores e medidas conforme a necessidade.

O admin.scss

Este arquivo vai englobar toda a área administrativa do site (e vai ser o arquivo básico que vamos usar para a maioria dos sistemas da Alego, já que eles são modelos administrativos).

Comentários iniciais

A primeira coisa que você vai fazer é editar o comentário inicial para identificar o arquivo em questão:

  1. Substitua o texto nome do sistema para o nome do sistema atual utilizando de caixa alta
  2. No campo Autor: coloque o seu nome
  3. No campo Data: coloque a data que iniciou o arquivo

Se quiser manter um changelog, separe com a mesma quantidade de traços (-) e siga a seguinte ordem:

  1. Separador de traços
  2. Título changelog: todo em maiúsculo
  3. Cada dia você coloca a data separa com um traço e coloca a descrição. As alterações mais recentes devem aparecer no topo e as mais antigas aparecem embaixo

Exemplo de changelog:

SASS

// -----------------------------------------------------------------------------

// CHANGELOG:

// 10/03/2016 - Esta foi minha última alteração

// 08/03/2016 - Esta foi a minha primeira alteração

// 05/03/2016 - Arquivo criado e finalizado

// -----------------------------------------------------------------------------

Cada entrada do changelog é depois que o arquivo já foi encerrado e serve para mostrar alterações depois que o sistema já estava pronto. Um exemplo de entrada: acrescentada formatação para a tela de registro de usuário ou unificadas classes da tabela x e tabela y.

Configurações iniciais

Depois dos comentários vem as configurações iniciais do arquivo.

A primeira configuração é a de importação de arquivos SASS. Por padrão já importa o arquivo base. Se você tiver arquivos específicos, inclua-os aqui. Em particular aqueles que você alterou alguma configuração padrão do FRONT-ALEGO e jogou na pasta extra (exemplo: você criou um modelo de tabela personalizada para usar com a classe .tab, então a importação seria @import "extra/tabela";).

Há uma exceção: arquivos de ajustes emergencial devem aparecer no final do arquivo, para que as configurações sobreponham a deste. Estes arquivos, no entanto, são temporários

Depois da importação está as configurações de grid (Veja mais detalhes aqui)l

Em seguida você coloca as variáveis que pertencem apenas a este arquivo. Isso só é relevante se você está trabalhando com mais de um arquivo diferente (como área administrativa e site). Algumas já foram definidas e indicam o tamanho da tela. Por padrão as classes devem aparecer em config/variaveis.scss.

As classes para serem usadas no login são:

.login
Fundo da tela de login. Utilize esta classe para definir a cor de fundo. Não faça isso no body pois este arquivo é herdado também para o resto do admin
.login__janela
A janela de login. Vem pré-configurada para ocupar a tela inteira em smartphone (de nada).
.login__header
Normalmente fica a imagem com o ícone e nome do sistema. Mas se quiser alterar o conteúdo fique à vontade.
.login__principal
A área principal da tela de login, onde ficam os formulários
.login__acoes
A área que fica o botão de logar e eventuais links de "esqueceu a senha" ou algo parecido

.field-12
Campo de texto que ocupa a largura toda (veja mais aqui)
.form-texto
Para formulários que tenham campo de texto (Veja mais aqui)
.bt
Ícone de botão (Veja mais aqui)

O admin é dividido em pedaços específicos, cada qual com sua classe (as classes são idênticas ao tag que as usa):

.header
O cabeçalho do sistema. Aqui vai ficar a marca, a indicação do nome do usuário e o botão de sair
.nav
A navegação do site. Contém o menu (mas pode conter algum outro elemento que se repete - como os dados de quem está logado, nome do sistema e assim por diante)
.main
A parte principal do site. Você pode incluir classes filhas com main__ ou sem o nome main para facilitar
.aside
Uma área extra. Pode ser área de banners, de relatórios. Esta tag/classe é opcional. Se não for usar delete o HTML e o CSS.
.footer
O rodapé do site, se houver. Esta tag/classe é opcional. Se não for usar delete o HTML e o CSS.

Configuração do menu

Esta configuração não aparece dentro do arquivo config.scss por se tratar de uma configuração específica para esta view (podemos ter um menu horizontal no site e vertical no admin, por exemplo).

$menu-tipo
Determina como o menu vai aparecer. Valores: vertical (padrão) e horizontal

Configurações de grid

O arquivo admin.scss contém toda a configuração de grid que está sendo usada. Conforme discutido na seção de grids não se usa classes próprias para grid e elas são inseridas diretamente no CSS. Neste caso fica no começo do arquivo.

$gd-header
A largura do header. O valor atual é $grid-qtd (a quantidade de grids definidas no arquivo de configuração)
$gd-menu
A largura do menu, que fica na lateral esquerda. O valor atual de exemplo é 2. Se o menu for horizontal ele é configurado automaticamente para $grid-qtd
$gd-aside
A largura da parte lateral. Se você não for usar o .aside basta inserir o valor 0
$gd-footer
O rodapé do site tem como valor $grid-qtd pois ocupa a largura toda
$gd-nav
A lagura da área de navegação. O valor depende se o menu é vertical ou horizontal. Valor automático, não há necessidade de alterar
$gd-main
A largura da parte principal. É uma conta que subtrai do valor total da grid o tamanho do menu e da parte lateral, para facilitar ajustes. Valor automático, não há necessidade de alterar

Variáveis

Algumas variáveis já foram aplicadas e você deve alterar conforme o sistema:

$header-altura
A altura do header. O valor atual é baseado na imagem de exemplo. Altere este valor conforme a necessidade do seu sistema.
$footer-altura-min
Regula a altura mínima do .footer. Se você não usar rodapé no sistema, coloque este valor como 0.
$main-altura-min
A altura mínima da área principal. Ela subtrai as duas variáveis acima (daí a importância de declará-las) e as margens da altura da tela para preencher o que falta na tela.

Formatações específicas

O CSS já vem pronto com algumas poucas formatações específicas. Algumas delas você formatou com as variáveis acima e outras formata posicionamento e grids. Cores de fundo, margens e espaçamento são sugestões.

As subclasses já estão aplicadas dentro delas (com o uso de &__filho).

Note, por exemplo, que o menu não tem fundo, bordas, etc. Ele tem apenas posicionamento e algum padding. O resto fica por sua conta.

Submenu

No exemplo já vem aplicado um submenu mais simples. Se o sistema que você não estiver trabalhando não tiver submenu, apague isso do admin.scss ou comente (caso queira ter submenu no futuro).

Muito cuidado com a estrutura. Se você quiser diferenciar o menu pai do menu filho, no seletor &__menu mude o elemento filho de li para > li.

Menu horizontal ou vertical

Por padrão o menu está na lateral esquerda. No entanto é possível fazer com que o menu fique horizontal. Para isso siga os passos:

  1. Altera a variável $menu-tipo de vertical para horizontal
  2. Acrescente uma variável de altura para o menu ($nav-altura).
  3. Acrescente - #{$nav-altura} na fórmula $main-altura-min. Com isso você está reduzindo a altura da parte principal com a inclusão do menu.
  4. Na classe .nav acrescente a altura criada anteriormente.
